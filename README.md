# 📁 파일 자동 분류 시스템

파일 이름과 내용을 분석하여 관련된 파일끼리 자동으로 폴더에 분류하는 Python 프로그램입니다.
> Python 3.8+ · Windows · 규칙 기반 + Claude AI 분류 · Dry Run / Undo 지원

---
## 1. 기획

바탕화면과 다운로드 폴더에 수많은 파일이 무분별하게 쌓이면, 필요한 파일을 찾기 어렵고 정리하는 데에도 많은 시간이 듭니다.  
이 프로젝트는 **파일 이름과 파일 내용을 함께 분석**하여 관련된 파일끼리 자동으로 분류해 주는 Python 프로그램을 만드는 것을 목표로 했습니다. :contentReference[oaicite:1]{index=1}

단순히 파일 확장자만 보고 정리하는 것이 아니라, 문서 내용 일부까지 읽어 의미를 파악하고, 보다 자연스럽고 실용적인 분류 결과를 제공하도록 설계했습니다.  
또한 실제 파일 이동 전에 결과를 확인할 수 있는 **미리보기 기능(`--dry-run`)** 과, 잘못 분류되었을 때 되돌릴 수 있는 **실행 취소 기능(`--undo`)** 을 포함하여 안전성도 확보했습니다. :contentReference[oaicite:2]{index=2}


## 📦 파일 구성

```
file_classifier/
├── main.py           # 메인 실행 파일
├── file_scanner.py   # 재귀 파일 탐색 및 메타데이터 추출
├── ai_classifier.py  # 파일 분류 엔진 (규칙 기반 + Claude AI)
├── folder_manager.py # 폴더 생성 및 파일 이동/복사, 롤백
└── reporter.py       # 결과 미리보기 및 완료 보고서 출력
```

## 🚀 실행 방법

### 기본 실행 (바탕화면 + 다운로드 폴더 자동 정리)
```cmd
python main.py
```

### 특정 폴더 지정
```cmd
python main.py --path "C:\Users\홍길동\Documents"
```

### 여러 폴더 동시 지정
```cmd
python main.py --path "C:\Users\홍길동\Documents" "C:\Users\홍길동\Desktop"
```

### 실제 이동 없이 미리보기만 확인
```cmd
python main.py --dry-run
```

### 파일을 이동 대신 복사
```cmd
python main.py --copy
```

### 마지막 작업 취소 (원래 위치로 복원)
```cmd
python main.py --undo
```

### Claude AI API 사용 (더 정확한 분류)
```cmd
set ANTHROPIC_API_KEY=your_api_key_here
python main.py
```

또는 실행 시 직접 지정:
```cmd
python main.py --api-key sk-ant-xxxx
```

### AI 없이 규칙 기반 분류만 사용
```cmd
python main.py --no-api
```

##. 특장점
1) 파일 이름 + 내용 기반 분석

단순 확장자 분류를 넘어서 문서 내용 일부까지 분석하므로 더 정확한 분류가 가능합니다.

2) Dry Run 지원

실제 파일을 이동하기 전에 결과를 미리 확인할 수 있어 실수를 줄일 수 있습니다.

3) Undo 지원

마지막 실행 결과를 기준으로 원래 위치로 복원할 수 있습니다.

4) AI 폴백 구조

Claude API가 없더라도 규칙 기반 분류로 기본 기능을 계속 사용할 수 있습니다.

5) 재귀적 폴더 탐색

하위 폴더까지 포함하여 정리할 수 있어, 중첩된 구조에도 대응할 수 있습니다.

6) 사용자 지정 경로 지원

기본적으로 Desktop / Downloads를 대상으로 하되, 임의 경로도 직접 지정하여 사용할 수 있습니다.

6. 한계점

현재 버전에는 다음과 같은 한계가 있습니다.

1) Undo가 1회 실행 기준

가장 마지막 작업에 대해서만 복원이 가능합니다.

2) 이미지/동영상 내용 분석 미지원

이미지와 영상은 현재 파일 내용 자체를 해석하지 못하고, 확장자 중심으로 분류됩니다.

3) 카테고리 구조 고정

기본 카테고리가 코드에 고정되어 있어 사용자가 자유롭게 수정하기 어렵습니다.

4) GUI 부재

현재는 커맨드라인 기반 프로그램이므로 초보 사용자에게는 진입장벽이 있습니다.

## 🔧 옵션 전체 목록

| 옵션 | 설명 |
|------|------|
| `--path 경로 [경로...]` | 분류할 폴더 (기본: 바탕화면+다운로드) |
| `--output 경로` | 결과 폴더 생성 위치 (기본: 바탕화면) |
| `--dry-run` | 미리보기만 (실제 이동 없음) |
| `--copy` | 이동 대신 복사 |
| `--undo` | 마지막 작업 취소 |
| `--no-api` | AI 없이 규칙 기반 분류 |
| `--api-key KEY` | Anthropic API 키 직접 입력 |
| `--min-files N` | 폴더 생성 최소 파일 수 (기본: 2) |

## ⚠️ 주의사항

- 실행 전 `--dry-run`으로 반드시 미리보기 확인을 권장합니다.
- 중요한 파일은 백업 후 실행하세요.
- `--undo`는 가장 마지막 1회 작업만 취소 가능합니다.
- 시스템 파일 및 숨김 파일(`.`으로 시작)은 자동으로 건너뜁니다.
