파일 자동 분류 시스템

AI 기반 파일 이름 및 내용 분석으로 자동 정리하는 Python 프로그램

언어 Python 3.8+
파일 수 5개 모듈
분류 방식 규칙 기반 + Claude AI
플랫폼 Windows
01

기획

어떤 문제를 왜 해결하려 했는가

바탕화면과 다운로드 폴더에 수백 개의 파일이 무분별하게 쌓이는 것은 대부분의 PC 사용자가 겪는 공통적인 문제입니다. 파일을 직접 하나씩 분류하려면 시간이 오래 걸리고, 분류 기준도 일관되지 않습니다.

이 프로젝트는 파일 이름과 파일 내용을 동시에 분석하여 관련된 파일끼리 자동으로 묶어주는 Python 프로그램을 개발하는 것을 목표로 했습니다. 실수를 방지하기 위한 미리보기 기능과, 잘못 이동된 경우를 대비한 되돌리기 기능까지 포함하여 안전성을 높였습니다.

특히 바탕화면·다운로드 폴더 외에도 사용자가 임의의 경로를 직접 지정할 수 있고, 하위 폴더부터 상위 폴더 순으로 처리하는 재귀 구조를 채택하여 중첩된 폴더 구조도 깔끔하게 정리할 수 있도록 설계했습니다.

🗂️
해결하려는 문제
수백 개 파일이 뒤섞인 바탕화면·다운로드 폴더를 수동으로 정리하는 비효율
🎯
핵심 목표
이름·내용 분석 기반 자동 분류, 미리보기·실행취소로 안전성 확보
👤
대상 사용자
파일 정리에 어려움을 느끼는 일반 Windows 사용자 전반
02

구성물

주요 기능 목록

총 5개의 Python 모듈로 구성되며, 각 모듈은 단일 책임 원칙에 따라 분리되어 있습니다.

file_classifier/
📁 file_classifier/
  ├── main.py            # 진입점 — 명령어 처리 및 전체 흐름 제어
  ├── file_scanner.py     # 재귀 탐색 — 하위→상위 파일 수집 및 내용 추출
  ├── ai_classifier.py    # 분류 엔진 — 규칙 기반 + Claude API 분류
  ├── folder_manager.py   # 실행 — 폴더 생성·파일 이동·롤백 로그
  └── reporter.py         # 출력 — 미리보기 트리 및 완료 보고서
SCAN
재귀 파일 탐색
os.walk(topdown=False)로 하위 폴더부터 순차 처리. 텍스트·PDF·docx 내용 앞 600자 샘플 추출. 숨김 파일·시스템 폴더 자동 제외.
CLASSIFY
AI + 규칙 기반 분류
13개 사전 정의 카테고리(확장자·키워드 가중치 기반). API 키가 있으면 Claude로 파일 내용까지 의미 분석. 파일명 토큰 유사도 보조 클러스터링.
PREVIEW
미리보기 출력
분류 계획을 트리 형태로 출력. --dry-run 옵션으로 실제 파일 이동 없이 결과만 확인 가능.
EXECUTE
폴더 생성 및 파일 이동
사용자 확인 후 바탕화면에 날짜별 폴더 생성. 이름 충돌 자동 처리. 이동 기록을 JSON 로그로 저장 → --undo로 원복.
03

실현 방법

사용 기술, AI 도구 활용 과정

순수 Python 표준 라이브러리(os, shutil, pathlib, argparse)를 중심으로 구현하여 별도 프레임워크 없이 Windows 환경에서 바로 실행 가능하도록 했습니다.

AI 분류는 Anthropic Claude API(claude-sonnet-4-20250514)를 활용했습니다. 파일 이름·타입·내용 샘플을 JSON으로 묶어 80개씩 배치 전송하고, 모델이 카테고리명과 설명을 JSON으로 반환하면 이를 파싱하여 폴더 구조에 반영합니다. API 키가 없는 환경에서는 확장자·키워드 가중치 기반 규칙 엔진으로 자동 폴백됩니다.

개발 전반은 Claude(Anthropic)와의 대화를 통해 계획 수립 → 모듈별 코드 작성 → 사용 설명서 자동 생성까지 진행했습니다.

🐍
Python 3.8+
표준 라이브러리 중심 구현. chardet, pypdf, python-docx 선택적 사용.
🤖
Claude API
Anthropic claude-sonnet으로 파일 의미 분석. API 없을 때는 규칙 기반 폴백 자동 적용.
📐
설계 원칙
단일 책임 원칙 5개 모듈 분리. bottom-up 재귀 탐색. JSON 롤백 로그.
🛡️
안전 장치
dry-run 미리보기, 사용자 확인 프롬프트, --undo 1회 복원, 이름 충돌 자동 번호 부여.

05

특장점

이 프로그램만의 강점

단순 확장자 분류를 넘어, 파일 내용까지 분석하는 다층적 접근으로 정확도를 높였습니다.

🔁 bottom-up 재귀 처리 하위 폴더 파일을 먼저 정리한 뒤 상위 폴더로 올라오는 구조로, 중첩 폴더도 빠짐없이 정리.
🛡️ dry-run 미리보기 실제 파일 이동 없이 분류 계획 전체를 먼저 확인 가능. 실수 방지의 핵심 기능.
↩️ 1회 실행 취소 JSON 로그 기반으로 마지막 작업을 취소하고 모든 파일을 원래 위치로 복원.
🤖 AI 폴백 구조 API 키가 없어도 규칙 기반 분류가 자동 활성화 — 오프라인에서도 완전히 동작.
📂 경로 자유 지정 바탕화면·다운로드 기본값 외에 여러 경로를 동시에 지정하여 한 번에 처리 가능.
📄 내용 기반 분류 텍스트·PDF·Word 파일의 앞부분을 읽어 이름만으론 파악하기 어려운 파일도 정확히 분류.
06

한계점

현재 버전의 한계

초기 버전으로서 아래와 같은 한계가 존재하며, 향후 개선 계획에 반영되었습니다.

⚠
되돌리기 1회 제한 현재 롤백은 가장 마지막 실행 1회만 지원. 여러 번 실행한 경우 이전 작업은 수동 복원 필요.
⚠
이미지·동영상 내용 분석 불가 이미지·영상 파일은 내용 분석 없이 확장자만으로 분류. 스크린샷과 사진이 같은 폴더로 묶일 수 있음.
⚠
분류 카테고리 하드코딩 13개 기본 카테고리는 코드에 고정되어 있어, 사용자가 직접 수정하려면 파이썬 파일을 편집해야 함.
⚠
GUI 없음 커맨드라인 전용으로, Python 및 명령 프롬프트에 익숙하지 않은 사용자에게는 진입 장벽이 존재.
